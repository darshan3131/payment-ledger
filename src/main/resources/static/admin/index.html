Architecture Notes

🔒 Optimistic Locking — Account entity uses @Version to prevent lost updates under concurrent writes

📤 Outbox Pattern — Events written atomically inside transaction, then published to Kafka by scheduler

🔁 Idempotency — Redis-backed key store prevents duplicate transaction processing

🔬 Reconciliation — Background job auto-heals PENDING transactions stuck for >5 minutes

📒 Double-Entry Ledger — Every transfer creates both a DEBIT and CREDIT ledger entry
